# GuitarBoard 项目实现文档

## 项目概述

GuitarBoard 是一个基于浏览器的**吉他指板音名记忆训练工具**。系统会在虚拟吉他指板上随机高亮一个位置（弦 × 品），用户需要从 12 个半音音名中选出正确答案。一轮完整练习包含 72 道题（6 弦 × 1-12 品），完成后展示正确率等统计信息。

## 核心模块详解

### 1. 音乐理论计算 — `src/lib/music.ts`

该模块是纯函数库，无任何副作用，负责所有音符与指板位置的计算。

**核心数据：**

- `NOTE_SEQUENCE`：12 个半音名 `['C', 'bD', 'D', 'bE', 'E', 'F', 'bG', 'G', 'bA', 'A', 'bB', 'B']`
- `STANDARD_TUNING`：标准吉他定弦 `['E', 'B', 'G', 'D', 'A', 'E']`（从第 1 弦到第 6 弦）

**核心函数：**

- `getNoteAtPosition(position)` — 根据弦索引和品号计算音名。算法：找到该弦空弦音在 `NOTE_SEQUENCE` 中的下标，加上品号偏移后取模 12。
- `createQuestion(position)` — 为指定位置生成一道题，包含唯一 ID（`crypto.randomUUID()`）、位置信息、正确答案。
- `createPracticeQuestions()` — 生成完整一轮 72 道题。先生成所有位置（6 弦 × 1-12 品），然后用 Fisher-Yates 洗牌算法打乱顺序。
- `getAccuracy(stats)` — 计算正确率百分比。
- `getInlayType(fretIndex)` — 返回品记号类型（3/5/7/9 品单点，12 品双点），用于在指板上渲染装饰点。

### 2. 音频系统 — `src/lib/audio.ts`

基于 `HTMLAudioElement` 的音频播放引擎，采用**两级缓存**策略：

- `soundCache`：缓存通用音效（`wrong`、`celebration`）
- `fretNoteCache`：缓存按位置索引的吉他音符采样（`/sounds/notes/string-{n}-fret-{f}.mp3`）

**关键设计：**

- 全局 `soundEnabled` 开关，由 UI 层通过 `setSoundEnabled()` 控制
- 播放前重置 `audio.currentTime = 0`，支持快速重复播放
- `play()` 调用包裹在 `try/catch` 中，优雅处理音频资源缺失
- 音符文件路径约定：`/sounds/notes/string-{1-6}-fret-{1-12}.mp3`

### 3. 国际化 — `src/lib/i18n.ts`

自研的轻量 i18n 方案，无需第三方库：

- `translations` 对象包含 `zh` 和 `en` 两个完整翻译字典
- `useTranslations(language)` 函数根据当前语言返回翻译对象
- 翻译覆盖所有 UI 文案，共 30+ 个翻译键

### 4. 类型系统 — `src/types.ts`

| 类型 | 说明 |
|------|------|
| `NoteName` | 12 个半音名的联合类型 |
| `FretPosition` | 弦索引 + 品号 |
| `Question` | 题目（id + 位置 + 正确答案） |
| `AnswerResult` | 答题结果（用户选择 + 正确答案 + 是否正确） |
| `PracticeStats` | 统计数据（总题数 + 正确数） |
| `GameStatus` | 游戏状态机：`idle → active → answered → completed` |
| `Translations` | 翻译字典的完整类型定义 |

## 组件架构

### App.tsx — 状态管理与游戏流程

`App` 是唯一有状态的组件，采用 React Hooks 管理全部游戏状态：

**状态：**

```
language: 'zh' | 'en'          // 当前语言
soundEnabled: boolean           // 音效开关
gameStatus: GameStatus          // 游戏状态机
currentQuestion: Question|null  // 当前题目
answerResult: AnswerResult|null // 当前答题结果
stats: PracticeStats            // 累计统计
questionQueue: Question[]       // 题目队列（72 道洗牌后的题）
currentQuestionIndex: number    // 当前题目在队列中的索引
```

**游戏流程：**

1. **idle** → 用户点击「开始练习」→ 调用 `createPracticeQuestions()` 生成洗牌后的题目队列 → 进入 `active`
2. **active** → 指板高亮目标位置 → 用户在 AnswerPanel 中点击音名 → 判断正误 → 播放对应音效 → 进入 `answered`
3. **answered** → 显示反馈横幅 → 若正确，1 秒后自动进入下一题；若错误，需手动点击「下一题」
4. 当题目队列为空 → 进入 **completed** → 播放庆祝音效 → 展示成绩总结卡片 → 可点击「重新开始」

**Effect 副作用：**

- 监听 `gameStatus` 变化，在完成时播放庆祝音效
- 监听 `soundEnabled` 变化，同步到 `localStorage` 和 audio 模块
- 监听 `answerResult`，正确作答后 1 秒自动跳转下一题

### Fretboard.tsx — 指板可视化

使用 CSS Grid 渲染 6 × 13 的指板网格（0 品 + 1-12 品）：

- 每根弦的线宽通过 CSS 变量 `--string-weight` 模拟真实弦的粗细（第 1 弦 1px → 第 6 弦 3px）
- 品柱使用 `::after` 伪元素绘制垂直线
- 弦线使用 `::before` 伪元素绘制水平线
- 目标位置根据答题状态显示不同样式：蓝色脉冲（未答）、绿色（正确）、红色抖动（错误）
- 品记号（inlay markers）在 3/5/7/9 品单点、12 品双点

### AnswerPanel.tsx — 音名选择

12 个音名按钮，排列为 flex wrap 布局。答题后根据结果为按钮添加状态样式：

- `is-correct`：用户选对 → 绿色高亮
- `is-incorrect`：用户选错 → 红色 + 抖动动画
- `is-reveal`：用户答错时，正确答案按钮以浅绿提示

### PracticePanel.tsx — 练习控制台

包含操作按钮和实时统计芯片：

- idle/completed 状态：显示「开始练习」/「重新开始」按钮
- active 状态：「下一题」按钮禁用（需要先作答）
- answered 状态：「下一题」按钮可用
- 统计展示：正确数/总数、正确率、当前目标位置

### LanguageToggle / SoundToggle — 辅助控件

- `LanguageToggle`：固定在右上角的中/英切换胶囊按钮
- `SoundToggle`：音效开关，状态持久化到 `localStorage`

## 测试策略

### 单元测试（`music.test.ts`、`audio.test.ts`）

- `music.test.ts`：验证音符计算、题目生成、去重、正确率
- `audio.test.ts`：验证路径生成、播放失败容错、全局开关、空弦跳过

### 集成测试（`App.test.tsx`）

使用 `@testing-library/react` + `userEvent` 模拟真实用户操作：

- 完整流程测试：开始 → 作答 → 自动跳题
- 错误作答：验证错误音效播放
- 语言切换：验证英文 UI 渲染
- 音效开关：验证状态持久化
- 练习完成：验证庆祝音效和总结卡片

### 测试基础设施（`test/setup.ts`）

- 用 `MockAudio` 替换全局 `Audio` 构造函数，避免真实音频播放
- `afterEach` 自动清理 mock、恢复定时器、重置音效开关、清理 DOM

## 设计亮点

1. **纯前端零后端**：所有逻辑在浏览器完成，无需服务端
2. **模块化分层**：UI 组件与业务逻辑完全分离，`lib/` 下为纯函数，易于测试
3. **Fisher-Yates 洗牌**：确保每轮练习顺序随机且均匀
4. **两级音频缓存**：避免重复创建 `Audio` 对象，提升播放性能
5. **CSS Grid 指板**：用纯 CSS 模拟真实吉他指板外观（弦粗细、品记号、品丝）
6. **无障碍支持**：ARIA 标签、`role="progressbar"`、`focus-visible` 样式
7. **优雅降级**：音频资源缺失时不崩溃，`try/catch` 静默处理
